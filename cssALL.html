Подключение стилей

Стили можно подключать 3-мя способами:

  • Если нужно подключить один тег к одному или двум стилям - это можно сделать прямо в html файле прописав style= внутри тега как атрибут
  • Можно подключать стили через специальный тег его нужно прописывать внутри тега хед. Внутри тега стайл мы прописываем все как и в отдельном файле с нашими ксс стилями (лучше всего код смотрится когда ксс в отдельном файле)
  • подключение стиля через отдельный файл. Для этого в теге хед мы используем такой тег как линк rel указываем как стайлщит, а ссылку - путь к нашему файлу

Селекторы для выборки элементов

Селектор - это элемент в нашем html файле который можно выбрать такими способами:

  • По тегу: скажем у нас будет уникальный тег в котором будет этот текст. В файле css мы указываем что это свойство будет присвоено именно этому тегу. Например прописав этот стайл для всего тега боди можно задать стиль всего сайта
  • По класу доступ в css - .название нашего класа{} . Класс может быть доступен для нескольких элементов и можно удобно задавать одинаковые свойства для этих элементов
  • По id - уникальному элементу на странице. доступ осуществляется через #наше значение айди

  • Доступ к тегу по названию. Для примера внутри нашего тега ли нужно создать еще один тег скажем Новая строка. Для доступа к этому тегу в css прописываем что наш тег п находится внутри другого тега в нашем случае можно просто указать ли потом п или юл ли п или юл п . В общем нужно указать условие в котром будет что наш тег находится внутри какого-то тега и не обязательно 1:2 , это может быть и 1:3 и т.д.

  • Доступ к этому абзацу будет осуществляться через тег у которого клас текст - смотрим css

  • Так же можно добавлять стили к нескольким обьектам одновременно используя , обычную запятую . Например мы хотим к класу три к тегу кью и айди identefecator добавить курсор граб - это рука при наведении мышкой

  • Допустим мы установили для всех тегов ли занчение текста красный, но для этого тега в котором мы сейчас находимся нам необходимо поставить значение - желтый.

    если мы обратимся с пробелом к класу внутри тега li .headyng то мы будем задавать стиль для наших обьектов которые находятся внутри, в нашем случае для обьекта п

    Для того чтобы доступ осуществлялся лишь к обьекту ли у которого клас хидинг (допустим что этот клас есть еще у див и гдето у п) нужно прописывать li.headyng без пробела
  • Вообще если нам нужно выбрать тег у которого айди или клас какой-то какой нам нужен а не обращаться к элементам внутри этого тега мы просто не ставим пробел между тегом и класом. И если наоборот нам нужно внутри тега див выбрать лишь какие-то элементы класса - мы обращаемся через пробел. Если нужно выбрать все элементы внутри какого нибудь тега или айди мы через пробел ставим * .
  • Есть одна очень интересная штука , можно делать селектор выбора скажем для тега i у которого предыдущий тег является тегом b . Выглядеть это будет вот так #bi b + i. Мы обращаемся к тегу i у которого предыдущий тег b и который находится внутри айди би
  • Выборка по родительскому элементу. Допустим нам нужно выбрать из дива в котором есть несколько параграфов и ссылок и есть ссылки внутри парагафов, нужно выбрать ссылки которые только непосредственно в диве а в парагфах не выбирать. Для этого нужно прописать div > a - позволяет выбрать лишь дочерний элемент. А наши ссылки внутри параграфов уже будут дочерними элементами параграфов
  • Выборка по атрибуту. li[title] по тегу ли мы выбираем тех у кого есть тайтл. и еще можно выборку сделать даже по атрибуту со значением li[title="выборка по атрибуту"] . Выбирать можно не только по атрибуту тайтл , а и по другим атрибутам (для ссылки например есть аттрибут таргет и т.д.)

Так же для многих шрифтов есть варианты импортировать разные жирности - их мы потом можем использовать в фонт вейте

Работа со шрифтами

  • Подключение шрифтов через скачивание
    Скачеваем и распаковываем наш шрифт в новую папку которую назовем к примеру fonts. Далее чтобы подключить шрифт заходим в css файл и прописываем такую конструкцию @font-face { в поле font-family: вводим название для нашего шрифта, которое мы можем придумать самостоятельно, но лучше всего не придумывать а брать оригинальное (может только немного сократить) название для этого шрифта; дальше необходимо указать путь к нашему шрифту через src: url() }

    Все, шрифт мы подключили теперь чтобы его использовать мы обращаемся через селектор к нашему абзацу font-family: и внутри пишем характеристику - название нашего шрифта , в нашем случае caveat

  • Подключение блоков через ссылку бывает двух видов:
    1.через link - просто копируем линк из нашего шрифта и вставляем его в тег хед
    2.через import - копируем все начиная с @ и вставляем в наш файл css
    Теперь чтобы нам вызвать наш шрифт обращаемся к характеристике font-family и устанавливаем значение - так как называется наш шрифт откуда мы его взяли или копируем прямо из нашей страницы где мы копировали линк и импорт

Псевдокласы и Псевдоэлементы

С помощью псевдокласов мы можем управлять различными свойствами и состояниями какого либо обьекта
Ссылка
В css мы прописали что наша ссылка будет синего цвета и без подчеркивания. С помощью псевдокласса мы сделаем так что при наведении ссылка будет менять цвет и будет подчеркнута (меняется ее поведение). Псевдокласы ставятся после селектора через одно двоеточие, а псевдоэлементы через 2 двоеточия

    Псевдокласы

  • :hover - псевдоклас который применяет стили при наведении мышки на этот обьект. И этот псевдоклас можно применять не только к ссылкам а и к другим тегам
  • :activ - будет срабатывать каждый раз когда мы будем нажимать на какой либо обьект
  • :visited - псевдоклас который применяется к ссылкам . Можем добавить к ссылкам по которыйм мы уже совершали клик
  • :link - наоборот к ссылкам на которые мы еще не переходили
  • :focus - не показывает фокус, а срабатывает тогда когда мы поставили курсор внутрь текстового поля , например когда мы что-то вводим , но когда мы выходим с этого поля - фокус перестает работать. Чтобы показать как это работает создаем импут или текстареа
    чтобы установить наш курсор не в самом начале строчки а сделать небольшой отступ - прописываем характеристику для импута text-indent: и указываем размер отступа в пикселях
    Чтобы убрать обводку вокруг нашего поля импут пользуемся такой функцией как outline: и устанавливаем занчение none. Так же когда буду делать конспект - изучить какие значения еще могут быть для аутлайн
  • :disabled - будет применено только к тем полям которые не активны
  • :enabled - будет применено только к тем инпутам которые активны
  • :required - будет применено к тем полям которые являются обязательными для заполнения
  • :empty - с помощью этого псевдокласа можно выбрать например абзацы у которых клас three , но которые являются пустыми

    :not(:empty) - наоборот выбрать не пустые

  • :first-of-type - например если выбрать этот псевдоклас к элементу скажем заголовок то мы можем играть со стилем именно первого заголовка
  • :last-of-type - то же самое только к последнему элементу выбраного типа
  • :last-child и :first-child последний и первый элемент в родительском классе
  • ul :nth-child(2) - доступ к второму элементу в додительском класе ul . Если надо взять только четные элементы вместо 2 пишем 2n или even. Если нужны нечетные числа 2n + 1 или odd. Каждое 4е число - 4n ну и так далее. Каждое 4е начиная со второго 4n + 1 и так далее
  • онли оф тейп ul p:only-of-type - позволяет выбрать единственный элемент (p) который находится в родительском классе (ul) . Если в нужном классе юл будет 2 параграффа - псевдоклас не сработает

    • Псевдоэлементы

    Псевдоэлементы дают возможность работать с какой-то частью текста внутри html обьекта. Псевдоэлементы прописуются через два двоеточия

  • ::after - выводит текст после элемента который необходимо прописать в атрибуте контент. Так же к этому тексту могут применяться различные стили
  • ::before - выводити кстати можно не только текст но и значение атрибута - скажем тайтл
  • ::first-letter - применяет стиль для первого символа внутри тега
  • ::first-letter - применяет стиль для первой линии внутри тега. Кстати прикольно что если делаешь масштаб и слова переносятся на вторую строчку - стиль остается таким же только для первой строчки - тоесть становится на меньшее количество символов

Работа с фоновыми картинками

для того чтобы установить задний фон для всего сайта лучше всего обращаться к тегу боди.

  • background-color можно применять не только в работе с фоновым рисунком , но и в обычных тегах - мы можем указать просто однотонный цвет
  • background-image - можно указать url() картинки которая будет использоваться для заднего фона нашего сайта
  • background-repeat: можно указать чтобы наша картинка фона
    • no-repeat не повторялась
    • repeat будет повторяться по координате x и y
    • repeat-x будет повторяться по горизонтальной линии
    • repeat-y будет повторяться по вертикали
  • background-position - позиция фоновой картинки на сайте
    • bottom - с самого низу по центру
    • center - тупо в самой средине сайта
    • inherit - в левом верхнем углу
    • initial - в левом верхнем углу
    • left - прижата к левому краю по средине сайта
    • right - прижата к правому краю по средине сайта
    • top - прижата к верхнему краю по средине сайта
    • unset - в левом верхнем углу
    • 100px - тупо по центру отступая от левого края на 100 пикселей
  • width - прописывать не стоит - сбивается все что мы делали до этого на сайте. Вообще штука запарливая эта бекграунд картинка , но может и стоит с ней разобраться
  • height работает
  • background-size - размер картинки
    • cover - картинка покрывает весь сайт
    • auto - ничего не поменялось
    • contain - занимает позцию по ширине экрана (при отдалении картинка занимает всю ширину экрана)
    • и естественно можно указать размер в пикселях и т.д.
  • background-attachment - то как ведет себя картинка когда мы листаем страницу
    • fixed - самая крутая штука. Когда мы листаем страничку наш бекграунд остается на месте и очень прикольно сайт начинает выглядеть. Если добавить фиксед к обьекту и листать сайт - получается очень прикольный эффект
    • local - не заметил изменений
    • scroll - не заметил изменений

position позиции обьектов

  • position: static - значение по умолчанию - ничего не меняется
  • position: relative - если просто задать позишн релатив - ничего не произойдет, но благодаря этому свойству мы можем двигать обьект относительно его местонахождения. У релатив есть 4 свойства. Позишн релатив приколен еще тем что другие обьекты думают что он остался на своем месте и обтекают его как и прежде. Таким образом можно немного сместив обьект добиться прикольного эффекта
    • left: 20px двигаем блок слева на право
    • right: 20px обьект справа на лево
    • top: 20px - двигаем обьект сверху вниз на 20 пикселей
    • bottom: -20px - двигаем обьект снизу вверх на минус 20 пикселей, тоесть тоже опускаем его на 20 пикселей
    • для позишн релатив существует еще такая штука как z-index: от 0 до бесконечности на запясти. Чем высше значение з индекса тем высше будет наш слой. Скажем если есть 2 блока красный(1) и синий(0) и они находят друг на друга то красный блок будет выше из за з индекса который больше
  • position: fixed - наоборот делает наш обьект как бы невидимым для других обьектов в плане обтекаемости и позволяет нам разместить этот обьект не относительно его местоположения, а относительно всего экрана. Этот блок сделан по принцыпу надоедливой рекламы на сайте - когда ты листаешь страницу вниз, а этот блок летает за тобой повсюду. Все 4 свойства из релатива тоже тут работают
  • position: absolut - работает точно так же как фиксед (распологая обьект относительно всей нашей странички, но в отличии от фиксед при пролистывании странички обьект остается наместе (вверху странички получается)
  • position: sticky - обьединяет в себе возможности фиксед и релатив. Обьект находится на своем месте на сайте. Мы изменяем его положение на сайте с помощью 4х свойств и когда мы пролистываем сайт - этот блок двигается вместе с нами создававя как-бы прикольную анимацию. Я кстати поместил этот блок внутри другого блока и прикольно что за поля этого блока наш стики не вылазит
  • Еще один лайфхак. Если в нашем сером большом блоке установить position: relative, то теперь внутри этого блока position: absolut будет работать нормально. Тоесть задавая обьекту pos :first-of-type значение position: absolute мы говорим о том что он будет перемещаться не относительно всего экрана, а относительно нашего большого серого блока и это уже намного лучше чем было до этого

Переопределение и значения по умолчанию

  • Допустим у нас есть параграф и мы задаем ему какие-то стили. Но внутри у нас есть строка и мы не хотим чтобы эта строка унаследовала стили для этого в значение характеристик элементов для этой строки мы устанавливаем такой параметр как initial.

  • >Оно возвращает значение которое было установлено по дефолту. Так же есть характеристики которые не наследуются, например отступы , бордеры и так далее. Мы можем сделать так чтобы наша строка унаследовала бордер от тега п при помощи значения inherit

  • То есть получается если мы не хотим чтобы элемент наследовал характеристики от родительского класса - используем initial, и наоборот если у нас характеристики не наследуются , а мы хотим чтобы наследовались - используем значение inherit

  • unset - установка значения по умолчанию , но если есть в родительском классе какое-то значение то оно устанавливает именно его

  • если нам нужно применить inherit initial unset ко всем характеристикам вместо отдельно к колор или бордер мы используем характеристику all: которая принимает только 3 этих параметра

  • допустим мы насоздавали кучу параграфов с кучей значений и установили значение через самую высокую приоритетность , но нам нужно одну характеристику поменять скажем для последнего тега п мы ее устанавливаем , но она не применяется. В таком случае нужно использовать такой лайфхак как !импотент. Он ставится после значения характеристики без пробела и говорит программе о том что эта характеристика и ее значение будет важнее всех приоритетов , к примеру color: yellow!important

Адаптивность и медиа запросы

Чтобы создать медиазапрос прописываем ключевое слово @media(max-width: 320px) {body {background: red;}} .Если ширина экрана будет меньше 320пикселей наш медиа запрос будет срабатывать. При нашем запросе будет срабатывать условие что для тега боди , задний фон будет становится красным. Максимальная ширина на который будет действовать этот стиль 320 пикселей. Все что высша этот стиль действовать не будет. Чтобы у нас адаптивность срабатывала при этом условии внутри тега хед обязательно нужно проставить мета тег который отвечает за адаптивность сайта << meta name="viewport" content="width=device-width, initial-scale=1.0" >>

    Бывает несколько медиазапросов
  • max-width: 600px - все что высше ширины экрана 600 пикселей медиа запрос применяться не будет
  • min-width: 1800px - все что меньше ширины экрана 1800 пикселей медиа запрос отображаться не будет
  • @media(max-width: 200px) and (min-width: 500px) - этот стиль будет срабатывать если наша ширин экрана будет от 200 до 500 пикселей

Так же если у нас мобильная версия сайта кардинально по стилю отличается от десктопной версии сайте нужно в теге линк где мы подключаем наш файл css со стилем прописать дополнительную характеристику медиа в которой указать что например если минимальная ширина сайта будет 900 пикселей то наш файл со стилями подключен не будет - << link rel="stylesheet" media="(min-width: 900px)" href="css/styles.css" >>
а как же теперь настроить стили для мобильной версии сайта ? все очень просто. Прописываем еще один линк и он будет срабатывать если не будет срабатывать условие медиазапроса << link rel="stylesheet" href="css/style-for-pos.css" >>
И таких линков может быть множество для адаптации под мобильные устройства или десктопы

Анимации

Чтобы создать анимацию в нашем css файле со стилями выбираем наш блок через селектор и указываем такую характеристику как animation-name: и придумываем ей имя. Дальше обязательно прописываем сколько будет длится наша анимация animation-duration: 7s;.
Можно так же указать animation-delay: - время задержки.
animation-iteration-count: - количество повторений нашей анимации , можно указать просо число или infinite (анимация будет зациклена и повторяться постоянно и идти от точки а в точку б)
animation-direction: - направление анимации которое бывает трех видов normal(из точки а в точку б), reverse(из точки б в точку а) и alternate и alternate-reverse(анимация будет зацыклена и будет идти из точки а в точку б, а потом из точки а возращаться на исходную позицию в точку б и опять идти к а)

дальше создаем собачку (там где мы прописываем обычно селектор) @keyframe имя анимации которое мы придумали и ставим наши скобки{}. Внутри мы указываем что будет происходить в начальной и конечной точке анимации (так же есть возможность делать множество из промежуточных точек).
Указываем какие характеристики у нас будут у анимации в начальной точке с помощью 0% или from и внутри {} прописываем наши начальные характеристики для анимации
промежуточные прописываем как 10% 40% и так далее. Ставим {} и прописываем внутри промежуточные характеристики
И в самом конце ставим to или 100% и указываем конечные характеристики для нашей анимации

Переходы

Чтобы создать эффект перехода, вы должны указать две вещи:
1.свойство CSS, к которому вы хотите добавить эффект
2.продолжительность эффекта
Примечание. Если часть длительности не указана, переход не будет иметь никакого эффекта, поскольку значение по умолчанию равно 0.
Эффект перехода начнется, когда указанное свойство CSS (ширина) изменит значение.
Теперь давайте укажем новое значение свойства ширины, когда пользователь наводит указатель мыши на элемент например был у нас блок шириной 100 пикселей , при наведении ширина становится 120 пикселей . для нашего блока тогда прописываем transition: all 0.6s ease; где в характеристиках указано олл - для всех характеристик будет применяться анимация , анимация будет длится 0.6 секунд , и переход будет как изи
transition: width 2s, height 4s - В следующем примере добавляется эффект перехода как для свойства ширины, так и для свойства высоты с продолжительностью 2 секунды для ширины и 4 секунды для высоты

transition-timing-function: Свойство задает кривую скорости эффекта перехода.
Свойство transition-timing-function может принимать следующие значения:
ease- указывает эффект перехода с медленным началом, затем быстрым, а затем медленным завершением (по умолчанию)
linear- задает эффект перехода с одинаковой скоростью от начала до конца
ease-in- указывает эффект перехода с медленным стартом
ease-out - указывает эффект перехода с медленным концом
ease-in-out - указывает эффект перехода с медленным началом и концом
cubic-bezier(n,n,n,n) - позволяет вам определять свои собственные значения в кубической функции Безье

transition-delay: 1s; - Свойство transition-delayуказывает задержку (в секундах) для эффекта перехода.Следующий пример имеет 1-секундную задержку перед запуском

Характеристики для наших блоков непосредственно

Какие мы свойства можем прописывать для каждого элемента ? Та впринцыпе как и в обычных характеристиках можно указывать все что захотим, но есть специальные характеристики флексбоксов:
flex-basis: - аналог width ширины который указывается в пикселях и т.д.
flex: - тоже параметр с помощью которого можно задавать ширину , но в этом параметре мы ее задаем с помощью коэфициэнтов. К примеру мы задаем для всех элементов флекс 1 - таким образом у нас все блоки будут равны между собой. Если для одного задем параметр флекс как 2 - он будет в 2 раза больше чем блоки с параметром 1 и т.д.
допустим что мы указали что все наши блоки распологаются в одну горизонтальную линию и распологаются по центру относительно самого высокого обьекта. А нам нужно использовать такое свойство как расположение одного из обьектов иначе
align-self: - работает непосредственно с нашим блоком и позволяет поменять конкретно его положение относительно остальных блоков
1. flex-start - просто прижимает к верху
2. flex-end - прижат к низу
3. и все остальные значения как для обычного родительского елайна

Характеристики для класса родителя - в котором расположены наши блоки

order: - позволяет сортировать элементы по порядку. По умолчанию ордер у всех 0. Чем меньше значение ордер (например -1) тем раньше он у нас будет отображаться и наоборот - блок с самым большим значением ордер будет в самом отображен последним

Так как по умолчанию флексбокс делает что все наши элементы не переностяся на новые ряды нам нужно в нашем главном классе Flexbox прописать такое свойство как flex-wrap:
1. wrap - позволяет переноосить элементы на новые ряды
2. nowrap - значение по умолчанию, все элементы будут идти в один ряд
3. wrap-reverse - переносим элементы на новые ряды но в обратной последовательности

justify-content: позволяет указать как будут распологаться наши блоки
1.flex-start - значение по умолчанию , блоки распологаются с начала строки
2. flex-end - все элементы будут прижаты к концу
3. center - расположение элементов ровно по центру
4. space-between - первый элемент будет при старте странички второй элемент будет либо ровно между 1 и 3 , но если 3й не помещается в этот ряд то второй будет прижат к концу (вправо), а третий будет начинатся с нового ряда с начала (слева)
5. space-around - расположение обьектов с равными отступами по левую и правую сторону. При том если 2 обьекта находятся на одной строке - то отступы получается как бы удваиваются (визуально, но фактически этот отступ первого и второго обьекта)

flex-direction: позволяет указать как именно будут распологаться элементы
1. row - значение по умолчанию, элементы распологаются в ряд
2. column - когда обьекты распологаться друг под другом
3. row-reverse - в один ряд, но в обратной последовательности
4. column-reverse - друг под другом в обратной последовательности

align-items: - позволяет разместить блоки по горизонтали
1. flex-start - значение по умолчанию , элементы прижаты к левому верхнему краю
2. stretch - наши блоки растягиваются на одну высоту и сохраняют свою ширину вне зависимости сколько контента внутри - просто увеличивается высота в таком случае
3. flex-end - прижаты к левому нижнему краю
4. center - просто по центру относительно самого высокого блока , между top и bottom
5. baseline - расположение относительно центральной горизонтальной линии
