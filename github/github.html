Установка и настройка

Для установки git его нужно скачать и потом работать в обычном терминале cmd или в терминале bash. Чтобы проверить версию и работу git нужно прописать в терминале git --version . Дальше нам необходимо создать репозиторий , для этого существует такая команда как git init . Важно ! Проэкт загружает в облако все файлы и папки внутри папки в которой открыт терминал и создает репозиторий. После выполнения этой команды создается внутри проэкта скрытая папка точкагит . Так же необходимо выполнить настройку аккаунта. Прописываем git config --global user.name 'maudzydun' . И Прописываем git config --global user.email 'semenzelenskij@gmail.com'

Добавление данных


добавление файлов в проэкт происходит с помощью команды git add (и здесь указываем что мы добавлем):

  • . - добавляем все файлы в репозиторий;
  • *.txt - добавление всех файлов с расширением например txt;
  • myimg/ - добавление папки например myimg;
  • myimg/*.html - добавление всех штмл файлов из папки myimg;
  • myimg/**/*.html - добавление всех штмл файлов из папки myimg и всех ее подпапок;
  • index.html - добавляет только один наш файл.

git rm --cached index.html - вот мы добавили в репозиторий какой-то файл но еще не закомитили его, и нам нужно его удалить чтобы при последующем комите этот файл не попал в репозиторий. Для этого пользуемся вот такой вот командой.


git commit или git commit -m 'тут указываем коментарий какие перемены у нас были сделаны' . При первом варианте у нас открывается специальное меню из которого чтобы выйти нужно initial commit как то сделать (от ситуации зависит). Во втором случае комит выглядит длинее но по факту происходит быстрее.


Игнорирование данных

В первую очередь создается файлик который начинается с точки и называется гитигнор .gitignore и внутри этого файла , как в блокноте прописываем какие файлы мы будем игнорировать. Каждый новый файл или папку мы прописываем с новой строки

.DS_Store # указанный файл в любой директории проекта
*.log # любой файл с указанным разрешением в любой директории проекта
**/*.log # аналогично правилу *.log
build/ # папка build по всему проекту
/build/ # папка build корневая директория
doc/*.html # все файлы html в директории doc
doc/**/*.html # все файлы html в директории doc и поддиректориях
!.gitignore # игнорирует исключение файла .gitignore
/packages/ # папка packages
!packages/config.xml # все файлы в папке кроме файла config.xml

Ветвление и Слияния

git branch forum - чтобы создать ветку в репозитории (например моя команда разработчиков занимается отдельно созданием форума на сайте)
git checkout forum - чтобы перейти на ветку по созданию форума нужно выполнить еще дополнительную команду
git merge forum - чтобы обьеденить ветки нужно перейти на вкладку мастер и прописать что мы хотим обьедениться с вкладкой форум и тогда произойдет это слияние

Удаленный репозиторий

Я подключился как-то интуитивно к git-hub исспользуя графический интерфейс VSCode и у меня был создан удаленный репозиторий с одноименным названием (как название папки в которой мы сейчас находимся). Но ниже приведу описание как правильно подключаться к удаленному репозиторию GitHub:
Первым делом нужно зайти на сайт GitHub и создать новый репозиторий. Далее гитхаб предлагает нам различные команды чтобы подключиться к этому репозиторию и выгрузить в него различные файлы.
echo "# www-app" >> README.md - эта команда создает эхо в терминале в которое записывается все что находится в двойных кавычках;
git init - дальше говорится о том что чтобы подключиться нужно инициализировать гит (мы это уже сделали ранее)
дальше по правилам нам нужно выгрузить все файлы проэкта add и все это дело закомитить (как мы это делаем постоянно)
git branch -M master - это говорит о том что нам нужно дальше создать ветку
git remote add origin https://github.com/maudzydun/www.git - непосредственно подключение к самому репозиторию
git remote - чтобы посмотреть к какому удаленному репозиторию мы подключены
git remote remove origin - удалить подключение к удаленному репозиторию
git push -u origin master - отправить данные с локального gitа на удаленный github
дальше идет авторизация на гит хаб (логин и пароль)
Дальше подключение все равно не произойдет так как у гит хаба настроена двойная аутентефикация. Чтобы это настроить нужно перейти на сайт github и зайти в настройки , пункт девелопер сетингс , девелоперс аксес токенс . Дальше надо нажать генерате нев токен , ввести пароль от гитхаба. Нужно ввести заметку , для чего будет использоваться этот токен (можно указать терминал , типо для авторизации в терминале). Как долго будет жить весь этот токен. Дальше выбираем для чего он будет использоваться (выбираем репо, так как будем использовать для репозиториевв). Дальше нажимаем генерате токен для его генерации. Дальше нужной найти учетные данные виндовс (пароли) в которых нужно найти гитхаб и вставить туда длиннющий пароль от токена
тут я завис и не нашел нужного меню поэтому по возможности подключаться к github через графический интерфейс
REDME.md - файл в котором мы должны описать наш проэкт . md - расширение для описания и в гугле можно прописать mark down и почитьать какие есть теги для этого расрешения
для того чтобы клонировать свой или чей-то репозиторий нужно желательно подняться хотябы на папочку высше "cd ../" и дальше нужно прописать git clone https://github.com/bogdandawabsheh/TCRC_DMS.git и вставить чейто гит . Дальше , если богдан сделал какие-то обновления в своем проэкте и мы хотим обновить наш проэкт то в таком случае нам нужно зайти в сам проэкт через cd и прописать в терминале команду git pull
